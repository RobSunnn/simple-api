API Documentation

Welcome to the API documentation page for our service. This API allows you to fetch interesting facts and more.

Authentication

All API requests require authentication. You need to pass a token in the header of your request.

Request Header

Authorization: Bearer <your-api-token>

Example (cURL)

curl -H "x-api-token: <your-api-token>" https://robsunnn-api.azurewebsites.net/fact

Token Request Example

If you need to obtain an API token, visit the Register page or log in if you already have an account.

Endpoints

GET /fact

This endpoint returns a random fact. You can optionally pass a query parameter for filtering facts.

Request Headers

  • x-api-token: Your API token (required)

Query Parameters

  • param: Optional query parameter for filtering facts.

Example Request

GET /fact?param=cat HTTP/1.1
Host: https://robsunnn-api.azurewebsites.net
x-api-token: <your-api-token>

Example Response

{
  "fact": "Cats sleep for 70% of their lives."
}

Error Handling

The API returns error responses in the following structure:

Error Response Example

{
  "status": 401,
  "message": "Invalid token",
  "error": "Unauthorized"
}

Common Errors

  • 401 Unauthorized: The token provided is invalid or missing.
  • 400 Bad Request: The request parameters are invalid or missing.
  • 500 Internal Server Error: An error occurred on the server.

Code Examples

Here are some quick code examples to help you get started:

JavaScript (fetch)

fetch('https://robsunnn-api.azurewebsites.net/fact', {
  method: 'GET',
  headers: {
    'x-api-token': ''
  }
})
.then(response => response.json())
.then(data => console.log(data));

cURL

curl -H "x-api-token: " https://robsunnn-api.azurewebsites.net/fact
